# STlink-V2.1烧录问题
#stm32 #stm32-F446RET6 #stm-ST-Link-V2_1 
1. 启动方式、复用等等，可能对烧录（甚至是cubeprg与目标板的链接）出现问题；甚至会出现刚刚连接上，但是过一会自动断连等问题![[Pasted image 20251102160853.png]]
2. 解决方式：
	1. 连接stlink，打开cubeprg，打开”固件升级“，![[Pasted image 20251102160934.png]]
	2. 进入页面，选择“Open in update mode”![[Pasted image 20251102160956.png]]
	3. 页面变动如下：自行刷新，并更新，这点不必多说；![[Pasted image 20251102161017.png]]
	4. 选择”Change Type“，一般stlink-v2.1到手上便是这个设置情况，“Mass storage”为勾选状态，即可在目标板之间传输文件；![[Pasted image 20251102161051.png]]
	5. 问题就出在这里，当选择了“Mass storage”之后，因为不是官方的板子，可能对引脚、资源等规划有异，便一般不能使用该模式进行连接；故，对于一般板子（不支持文件传输功能的），做出如下改动，勾选“STM32 Debug+VCP”![[Pasted image 20251102161143.png]]
	6. 点击“Upgrade”，进行配置更新![[Pasted image 20251102161259.png]]
	7. 进度完成，显示如下![[Pasted image 20251102161321.png]]
	8. 返回cubeprg，把stlink-v2.1拔下再插上，点击“Connect”，进行连接![[Pasted image 20251102161426.png]]
	9. 成功
