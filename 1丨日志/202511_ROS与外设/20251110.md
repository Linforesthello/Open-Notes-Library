## 🧩 一、为什么两个问题一起好了？

|问题|根因|解决原理|
|---|---|---|
|🔴 `#include "ros/ros.h"` 找不到|VSCode 打开错层级，`${workspaceFolder}` 错误，IntelliSense 搜不到 `/opt/ros/noetic/include`|从 `catkin_ws` 顶层打开后，`c_cpp_properties.json` 自动生效，路径恢复正常|
|🐍 PlatformIO / Python 报 “找不到解释器”|VSCode 在包目录下找不到全局 `.vscode/settings.json` 与 `pythonPath`|工作区层级正确后，VSCode 找到了主环境 `/usr/bin/python3`，Python 插件配置同步|

换句话说：

> **VSCode 以为你在一个“孤立文件夹”里开发，但你其实在一个完整的 ROS 工程中。**

而当你切回到正确的工作区后：

- 所有 `.vscode` 配置（C++ / Python / PlatformIO）都能生效；
    
- 所有路径（`devel/include`、`src/**`、`/opt/ros/noetic/include`）都能被识别；
    
- 所有 ROS 工具（catkin、cmake、roslaunch）都能正确索引环境变量。

## 🧠 二、以后要记住的小经验

|项目类型|正确打开方式|
|---|---|
|ROS 工程|打开 `catkin_ws` 顶层文件夹|
|STM32 工程（Keil / CubeIDE）|打开 `.project` 或 `.ioc` 文件所在目录|
|PlatformIO 项目|打开 `platformio.ini` 所在文件夹|
|Python / Conda 工程|打开含 `environment.yml` 或 `.venv` 的目录|

📌 原则一句话：

> **永远从项目的“根目录”打开，而不是子文件夹。**
