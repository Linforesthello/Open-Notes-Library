### **标题：在手机端使用 Termux 与 Obsidian 管理 Git 仓库的完整实践与问题解决**

---

### **总结内容**

1. **Termux 基础与环境配置**
    
    - Termux 是 Android 上的类 Linux 终端环境，可用来运行 Git、SSH 等命令行工具。
        
    - 配置 SSH 密钥（`ssh-keygen -t ed25519`）以便免用户名密码推送 GitHub 仓库。
        
    - 使用 `termux-setup-storage` 授权访问手机存储，挂载 `~/storage`。
        
    - 遇到的常见问题：`error: unable to open file "/data/data/com.termux/files/usr/bin/sh"`，以及 dubiuous ownership，需要用 `git config --global --add safe.directory <path>` 解决。
        
2. **Git 与 GitHub 操作流程**
    
    - 初始化仓库：`git init`
        
    - 设置分支：`git branch -M main`
        
    - 添加远程：`git remote add origin <url>`
        
    - 推送和拉取：
        
        - 日常操作流程：`git add .` → `git commit -m "msg"` → `git pull --rebase` → `git push`
            
    - 设置分支跟踪远程：`git branch --set-upstream-to=origin/main main`
        
    - 遇到合并冲突：手动解决冲突文件后使用 `git add <file>` → `git rebase --continue`
        
    - 强制同步远程（覆盖本地）：`git fetch origin && git reset --hard origin/main`
        
3. **Obsidian 与 Git 集成问题**
    
    - Obsidian Git 插件在 Android 上不稳定，可能报错：
        
        - `"Git remote ... uses an unrecognized transport protocol: 'ssh'"`
            
        - `"The function requires a "remote OR url" parameter but none was provided."`
            
    - 解决方案：**Obsidian 仅用于编辑笔记，Git 操作全部通过 Termux 完成**
        
    - 避免本地配置文件冲突：
        
        - 将 `.obsidian/` 文件夹加入 `.gitignore`
            
        - 删除已上传的 `.obsidian` 文件夹：
            
            ```bash
            git rm -r --cached .obsidian
            echo ".obsidian/" >> .gitignore
            git add .gitignore
            git commit -m "Remove .obsidian folder from repo and ignore it"
            git push
            ```
            
4. **处理 Git 合并冲突**
    
    - 使用 `git status` 查看冲突文件
        
    - 对于二进制文件或 Obsidian 配置文件，通常选择保留远程版本或手动备份
        
    - 使用 `git restore --staged <file>` 撤销暂存，`git add <file>` 标记冲突已解决
        
5. **清理远程仓库中的临时或系统文件**
    
    - `.trash/` 文件夹：系统或 Obsidian 的回收站，不必上传
        
    - 删除方法：
        
        ```bash
        git rm -r --cached .trash
        echo ".trash/" >> .gitignore
        git add .gitignore
        git commit -m "Remove .trash folder and ignore it"
        git push
        ```
        
6. **日常 Git 使用最佳实践**
    
    - 在 Termux 中执行完整 Git 流程，Obsidian 仅用于编辑
        
    - 保持仓库干净，忽略本地配置文件和临时文件
        
    - 使用 SSH Key 实现免用户名密码推送
        
    - 遇到冲突或远程更新时，先拉取再推送，必要时使用 `rebase`
